# Agent Instructions

## Commit Requirements

- Every non-release commit must update `docs/src/content/docs/changelog.md` in the same commit.
- Changelog updates must use the next release number and describe the bugs fixed or features added.
- Do not leave relevant documentation updates for a later commit. Include docs changes in the same commit as the behavior or code change they describe.
- Keep automated release commits limited to release metadata when possible; feature, fix, test, and docs commits should carry their own changelog and documentation updates.
